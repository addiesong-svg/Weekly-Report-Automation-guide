
# 📋 주간 업무 보고 자동화 설치 가이드

> **(방법1) Cowork에서 'https://github.com/addiesong-svg/Weekly-Report-Automation-guide' 설치해줘 합니다.**

> **(방법2) 이 파일을 Cowork에서 Claude에게 보여주고 "주간보고 자동화 세팅해줘"라고 말하면 Claude가 아래 과정을 자동으로 진행합니다.**

---

## 🤖 Claude에게 (자동화 세팅 지침)
 
아래는 **Claude가 팀원의 주간 보고 자동화를 세팅할 때 따라야 할 전체 지침**입니다.
 
---
 
### STEP 1. 개인 설정 수집 (AskUserQuestion 도구로 순서대로 질문)
 
사용자에게 다음 항목을 **하나씩** 질문하여 값을 수집하세요:
 
1. **이름 (한글)** — 팀즈 검색 키워드로 사용됩니다.
   예시: `홍길동`
 
2. **슬랙 User ID** — 슬랙 프로필 → `…` 메뉴 → "멤버 ID 복사"에서 확인 가능합니다.
   예시: `U04VBGU8Z9D`
   *(모르면 이름을 알려주면 Claude가 대신 찾아드립니다.)*
 
3. **업무용 이메일 주소** — 아웃룩 수신 메일 필터링에 사용됩니다.
   예시: `hong@nexon.co.kr`
 
4. **노션 DB 주소** — 주간 보고를 저장할 노션 데이터베이스 URL
   예시: `https://www.notion.so/nexoncompany/xxxx`
 
5. **카테고리 이름 2개** — 업무를 분류할 프로젝트/서비스 이름
   예시: `NP`, `바람:연`
   *(1개만 있거나 다른 기준이 있으면 그대로 알려주세요.)*
 
6. **자동 발송 제외 이메일 제목** — 매주 자동으로 오는 리포트 메일 중 요약에서 빼고 싶은 항목
   예시: `주간 유저 현황 리포트`, `KPI 자동화 리포트`
   *(없으면 "없음"이라고 답해주세요.)*
 
7. **슬랙 채널 목록** — 업무에서 주로 사용하는 슬랙 채널명을 카테고리별로 알려주세요.
   (스크린샷 공유도 가능합니다.)
 
8. **팀즈 채팅/채널 목록** — 업무에서 주로 사용하는 팀즈 채팅방/채널명을 알려주세요.
   (스크린샷 공유도 가능합니다.)
 
---
 
### STEP 2. 슬랙 User ID 자동 조회 (사용자가 모를 경우)
 
사용자가 슬랙 User ID를 모른다고 하면, `slack_search_users` 도구로 이름을 검색하여 User ID를 찾아주세요.
 
---
 
### STEP 3. CLAUDE.md 규칙 파일 저장
 
아래 경로에 개인화된 규칙 파일을 저장하세요:
 
```
/sessions/[현재 세션 ID]/mnt/.claude/projects/-sessions-[현재 세션 ID]/CLAUDE.md
```
 
저장할 내용 형식:
 
```markdown
# [이름] 주간 업무 요약 자동화 규칙
 
## 1. 검색 방법
 
### 슬랙
- 반드시 User ID로 검색
- User ID: [수집한 User ID]
- 검색 쿼리: from:<@[User ID]> after:YYYY-MM-DD before:YYYY-MM-DD
 
### 팀즈
- sender 이메일 필터 불가 → 이름(한글) 키워드로 검색
- 검색 키워드: [이름]
 
### 아웃룩
- sender 필터 사용: [이메일 주소]
 
---
 
## 2. 포함 기준
 
1. 내가 직접 작성한 메시지/이메일
2. 상대방이 나를 태그한 메시지 (슬랙, 팀즈)
3. 메일 수신자가 나인 경우 (아웃룩)
 
**제외 기준 (중요)**
- 내가 CC로만 포함된 스레드 (주도자가 타인인 경우) → 포함 금지
- 타인이 주고받는 대화에 단순 참조된 경우 → 포함 금지
 
---
 
## 3. 제외 항목
 
[수집한 제외 이메일 제목 목록]
 
---
 
## 4. 작성 형식
 
- 대상: 상급자에게 보고하는 주간 보고
- 구조: 번호. **굵은 제목** + 불릿 포인트(•) 상세 내용
- 금지: 모호한 표현 ("해당 팀" 등) → 반드시 구체적인 팀명 사용
- 특정 인물 실명 언급 금지
 
---
 
## 5. 카테고라이징
 
[카테고리 1]과 [카테고리 2]로 분리하여 정리
 
**슬랙 채널 목록**
 
[카테고리 1] 채널:
- [수집한 카테고리 1 슬랙 채널 목록]
 
[카테고리 2] 채널:
- [수집한 카테고리 2 슬랙 채널 목록]
 
**팀즈 채팅/채널 목록**
 
[카테고리 1] 관련 채팅:
- [수집한 카테고리 1 팀즈 채팅/채널 목록]
 
[카테고리 2] 관련 채팅:
- [수집한 카테고리 2 팀즈 채팅/채널 목록]
 
**팀명 카테고리 매핑 (중요)**
- 팀명이 어느 카테고리 소속인지 불분명할 경우, 채널/이메일 출처 기준으로 판단
- 특정 팀이 특정 카테고리에만 속한다고 사용자가 명시한 경우 반드시 그 카테고리에만 포함
 
---
 
## 6. 노션 DB 입력
 
- 내용 칸: 제목이 아닌 실제 업무 요약 텍스트 입력
- 다중 선택: [카테고리 1] 또는 [카테고리 2] 태그
- 날짜: 해당 주 월~일 범위 입력
- 노션 DB URL: [수집한 노션 DB 주소]
```

### STEP 4. 노션 DB 구조 확인 및 data_source_id 조회
 
`notion-fetch` 또는 `notion-query-data-sources` 도구로 사용자가 제공한 노션 DB URL에 접근하여 `data_source_id`를 확인하고 CLAUDE.md에 추가 저장하세요.

### STEP 5. 스케줄 태스크 2개 생성
 
#### 태스크 1 — 요약 생성 및 DM 발송
`create_scheduled_task` 도구로 매주 금요일 오후 6시 실행 태스크를 등록하세요.
 
```
taskId: weekly-report-[이름 영문 or 닉네임]
cronExpression: 0 18 * * 5  (매주 금요일 오후 6시)
description: [이름] 주간 업무 요약 — 슬랙/아웃룩/팀즈 수집 후 서브 에이전트 검증 및 본인 DM 발송 (노션 저장은 :okk: 승인 후 처리)
```
 
태스크 프롬프트에는 아래를 포함하세요:
- 슬랙/아웃룩/팀즈 수집 → 요약 작성 → 서브 에이전트 검증
- **노션에는 저장하지 않고** 본인 DM으로 발송
- DM 메시지에 `[주간보고_검토요청_YYYYMMDD]` 마커 포함
 
#### 태스크 2 — 승인 체크 및 노션 저장
`create_scheduled_task` 도구로 금요일 오후 6~23시 매시간 체크 태스크를 등록하세요.
 
```
taskId: weekly-report-approval-check-[이름 영문 or 닉네임]
cronExpression: 0 18-23 * * 5  (매주 금요일 오후 6~23시 매시간)
description: [이름] 주간보고 승인 체크 — DM의 :okk: 반응 감지 시 노션 DB에 저장
```
 
태스크 프롬프트에는 아래를 포함하세요:
- 본인 DM에서 `[주간보고_검토요청_]` 마커가 있는 최근 메시지 탐색
- `:okk:` 이모지 반응 여부 확인
- 반응 있으면: 기존 노션 항목 아카이브 후 새 항목 저장
- 완료 후 DM 스레드에 `✅ 노션에 저장 완료되었습니다! [노션저장완료]` 답장
 
태스크 프롬프트에는 아래 전체 워크플로우를 포함하세요.
 
---
 
### STEP 6. 자동화 실행 워크플로우 (매주 금요일 실행 내용)
 
스케줄 태스크가 실행될 때마다 아래 순서로 수행합니다:
 
#### 6-1. 이번 주 날짜 범위 계산
- 이번 주 월요일(after) ~ 금요일(before) 날짜 자동 계산
 
#### 6-2. 슬랙 수집
- `slack_search_public_and_private` 도구 사용
- 쿼리: `from:<@[User ID]> after:[월요일] before:[금요일]`
- DM, 그룹DM, 일상 잡담은 제외하고 **업무 채널 내 실질 업무 메시지**만 포함
- 추가로 `<@[User ID]>` 태그된 메시지도 수집 (타인이 태그한 경우)
 
#### 6-3. 아웃룩 수집
- `outlook_email_search` 도구 사용
- `from:[이메일]` 또는 `to:[이메일]` 필터
- 제외 목록에 해당하는 이메일 제목은 자동 스킵
 
#### 6-4. 팀즈 수집
- `chat_message_search` 도구 사용
- 키워드: `[이름]`
- sender 이메일 필터 사용 불가 → 키워드 검색 후 발신자 이름으로 본인 메시지 필터링
 
#### 6-5. 요약 작성
 
수집된 내용을 바탕으로 아래 형식으로 작성:
 
```
[카테고리 1] 업무 요약 (YYYY-MM-DD ~ YYYY-MM-DD)
 
1. **[업무 제목]**
   • [상세 내용]
   • [상세 내용]
 
2. **[업무 제목]**
   • [상세 내용]
 
[카테고리 2] 업무 요약
 
1. **[업무 제목]**
   • [상세 내용]
```
 
#### 6-6. 서브 에이전트 검증
 
별도 서브 에이전트를 실행하여 다음을 검토:
1. **누락 확인**: 수집된 원본 데이터와 요약본을 비교하여 빠진 업무 항목 체크
2. **품질 검토**: 모호한 표현, 실명 노출, 맥락 없는 문장 여부 확인
3. 문제 발견 시 수정 후 최종본 확정
 
#### 6-7. 노션 DB 적재
 
`notion-create-pages` 도구로 각 카테고리별 1개 항목씩 DB에 입력:
- `내용` 칸: 요약 텍스트
- `다중 선택` 칸: 카테고리 태그
- `날짜` 칸: 해당 주 날짜 범위
 
---
 
### STEP 7. 완료 안내

## ❓ 자주 묻는 질문
 
**Q. 슬랙 User ID를 어디서 찾나요?**
A. 슬랙에서 내 프로필 클릭 → `…` 더보기 → "멤버 ID 복사" 를 누르면 됩니다. 모르면 Claude에게 이름을 알려주면 찾아드립니다.
 
**Q. 노션 DB 주소를 어떻게 확인하나요?**
A. 노션에서 해당 데이터베이스 페이지를 열고 우측 상단 `공유` → `링크 복사`하면 됩니다.
 
**Q. 카테고리가 1개이거나 다른 기준이 있으면?**
A. Claude에게 원하는 분류 기준을 자유롭게 말씀해 주세요. 맞게 적용해 드립니다.
 
**Q. 중간에 설정을 바꾸고 싶으면?**
A. Claude에게 "주간보고 자동화 설정 수정해줘"라고 말하면 재설정이 가능합니다.
